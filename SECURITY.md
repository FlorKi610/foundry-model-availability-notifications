# Security Policy

## Supported Scope

This repository is intended to stay private and focused on the model availability tracking features only.

## Reporting

Do not disclose potential vulnerabilities publicly.

Report security issues privately through the private repository owner workflow or GitHub private reporting if enabled on the repository.

## Operational Baseline

- Keep the repository private.
- Enable branch protection on `main`.
- Enable secret scanning and dependency alerts in GitHub settings.
- Use least-privilege repository access.
- Store credentials only in GitHub Actions secrets, never in tracked files.
